Since we are interested in the cloze probability in each condition, we need to calculate that, too. For this, we’ll create a new dataframe called grouped_data because crucially, we are grouping all of our data by Item and Condition. Notably, this means we won’t have random effects by Subject, only by Item. grouped_data has the following columns Item: the item number cp1: the encoded clause placement level l1: the encoded location level Cloze Prob: the cloze probability of the target word for each item and condition

grouped_data <- data %>% 
  group_by(Item, cp1, l1) %>%
  summarise(ClozeProb = sum(ResponseIsTarget) / length(ResponseIsTarget), 
            .groups = 'keep',
            across(.cols=c('Condition','ClausePlacement','Location'))
            )

LMM model Now we are ready to create our models As mentioned before, we won’t have any random effects by Subject, only by Item.

lm_model1 <- lmer(ClozeProb ~ 1 + cp1 * l1 +
                   (1 + cp1 + l1 + cp1 * l1 || Item),
                 REML=FALSE, 
                 data = grouped_data,
                 control = lmerControl(calc.derivs=FALSE), 
                 na.action = na.omit
                 )

An alternative: GLMM Here, our response variable is just ResponseIsTarget, a binary variable. We can use random effects by Subject since we aren’t grouping our data by Item and Condition as with the LMM. However, this model failed to converge with random effects on the interaction, so we had to remove it.
